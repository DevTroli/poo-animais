## 🌿 Como Contribuir com Branches

Para evitar conflitos, **cada integrante trabalha na sua própria branch** e só integra o código à `main` via Pull Request.

---

### 🔧 Passo 1 — Clone o repositório *(só na primeira vez)*

```bash
git clone https://github.com/usuario/nome-do-repositorio.git
cd nome-do-repositorio
```

---

### 🌱 Passo 2 — Crie sua branch

Uma branch é uma **cópia paralela do projeto** só sua, onde você trabalha sem afetar o código dos outros.

Use o nome da sua branch conforme a tabela acima:

```bash
# Exemplos — use o nome correspondente ao seu
git checkout -b animal/ryan
git checkout -b animal/gabriel
git checkout -b animal/pedro
git checkout -b animal/leonardo
git checkout -b animal/matheus
git checkout -b animal/pablo
```

> Isso cria e já te coloca dentro da nova branch automaticamente.

---

### 💻 Passo 3 — Desenvolva suas classes

1. Crie seus arquivos `.java` dentro de `src/br/edu/fatecpg/abstraçao/model/`
2. Implemente os **5 atributos** e **3 métodos** conforme a tabela do seu nome acima
3. No `Main.java`, instancie seus objetos:

```java
// Exemplo — adapte para os seus animais
Leao leao = new Leao();
leao.rugir();

Tigre tigre = new Tigre();
tigre.nadar();
```

---

### 💾 Passo 4 — Salve seu progresso com commits

```bash
git add .
git commit -m "✨ feat: adicionar classes Leao, Tigre e Elefante"
```

> Dicas de mensagens de commit:
> - `feat: cria classe Coruja com atributos e métodos`
> - `fix: corrige método hibernar do Urso`
> - `chore: instancia animais do Ryan no Main`

---

### 📤 Passo 5 — Envie sua branch para o GitHub

```bash
git push origin animal/ryan   # substitua pelo nome da SUA branch
```

---

### 🔁 Passo 6 — Abra um Pull Request (PR)

#### O que é um Pull Request?

Um **Pull Request (PR)** é um pedido formal para que o seu código seja **revisado e integrado** à branch `main`. É como dizer: *"Terminei minha parte — alguém revisa e aceita?"*

Ele serve para:
- 👀 **Revisão de código** — outros integrantes podem ver e comentar o que você fez
- 🐛 **Identificar erros** antes de misturar ao código principal
- 🤝 **Colaboração organizada** — tudo fica registrado com histórico

#### Como abrir um PR:

1. Acesse o repositório no **GitHub**
2. Você verá um aviso amarelo: *"Compare & pull request"* — clique nele
3. Preencha:
   - **Título**: `feat: adiciona animais do Ryan`
   - **Descrição**: liste brevemente o que você implementou
4. Clique em **"Create pull request"**

---

### ✅ Passo 7 — Revisão e Merge

- Um colega (ou o responsável) revisa o PR
- Se estiver tudo certo, clica em **"Merge pull request"**
- Suas classes entram oficialmente na `main` 🎉

```
  │
  ├── animal/ryan       ← Leão, Tigre, Elefante

  ├── animal/pedro      ← Coruja, Pinguim, Urso
  ├── animal/leonardo   ← Tubarão, Golfinho, Águia
  ├── animal/matheus    ← Cachorro, Gato, Cavalo
  └── animal/pablo      ← Macaco, Girafa, Jacaré, Camelo

Cada um termina → abre PR → colega revisa → merge na main ✅
```

---

## ⚠️ Regras Importantes

- ❌ **Nunca** commite diretamente na branch `main`
- ❌ **Nunca** edite o arquivo de outro integrante sem combinar antes
- ✅ Sempre crie sua branch a partir da `main` atualizada
- ✅ Antes de criar sua branch, rode: `git pull origin main`

---

## 🆘 Comandos Úteis

| Comando | O que faz |
|--------|-----------|
| `git status` | Mostra arquivos modificados |
| `git branch` | Lista todas as branches locais |
| `git checkout main` | Volta para a branch main |
| `git pull origin main` | Atualiza sua main com o repositório remoto |
| `git log --oneline` | Histórico de commits resumido |
